System.out — cómo hablarle al usuario
println · print · printf
System.out.println("Hola");      // + salto
System.out.print("Sin salto "); // sin salto
System.out.print("línea\n");
System.out.println();             // línea vacía

System.out.printf(
  "Hola, %s. Puntos: %d%n", nombre, pts
);
Secuencias de escape
"\n"   → salto de línea
"\t"   → tabulación
"\""   → comilla doble literal
"\\"   → barra invertida
"%n"   → salto en printf (portable)
"%%"   → signo % literal en printf
println vs. printf: println imprime y agrega salto automáticamente. printf formatea con especificadores y no agrega salto — hay que escribir %n explícitamente.

var · final · Tipos por referencia
var — inferencia de tipo (Java 10+)
var ciudad = "Bogotá"; // String
var nivel  = 5;        // int
var saldo  = 1500.75;  // double

// tipo fijo en compilación:
// ciudad = 99; ← NO compila
final — constantes
final double IVA = 0.19;
final int MAX_INTENTOS = 3;

// IVA = 0.21; ← error:
// cannot assign to final variable

System.out.printf(
  "IVA: %.0f%%%n", IVA * 100
);
Tipos por referencia: Todo lo que no es primitivo — String, colecciones, tus propias clases — es un tipo por referencia. La variable guarda una dirección a un objeto en memoria, no el valor directamente. Los verán en profundidad en OOP.

Scanner · Leer input del usuario

Importar, declarar, leer
— y no olvidar el buffer.

Uso básico
import java.util.Scanner;

Scanner sc = new Scanner(System.in);

System.out.print("Tu nombre: ");
String nombre = sc.nextLine();

System.out.print("Tu edad: ");
int edad = sc.nextInt();

sc.close();
⚠️ La trampa del buffer
nextInt() lee el número pero deja el \n en el buffer. El siguiente nextLine() captura ese \n vacío en lugar de esperar al usuario.
int puntos = sc.nextInt();
sc.nextLine(); // limpia el \n sobrante
String ciudad = sc.nextLine(); // ahora sí
Aplica después de nextInt(), nextDouble() y nextFloat() cuando le sigue un nextLine().
